Exploring Proxmox and Setting Up a Home Server

I'm currently diving into the world of Proxmox and setting up a home server to enhance my understanding of server management and security practices. One of objectives was to segregate my network LAN for improved security.

To fortify the server which is a type1 virtualized Ubuntu, I've brought a combination of security measures, utilizing iptables, chroot, seccomp, and AppArmor. These tools play a crucial role in hardening the server environment, ensuring a robust defense against potential vulnerabilities.

As well some ACL with users of least privilege principle, In my pursuit of a secure setup, I've integrated Snort for comprehensive logging capabilities and alerts to notify me to take precautions. Snort also allows me to monitor network traffic and detect potential threats, providing an additional layer of security to the server. To counteract brute force attacks, I've implemented Fail2Ban combined with iptables, a proactive security measure that automatically blocks malicious actors attempting unauthorized access.

Moreover, I've established rules to prevent Distributed Denial of Service (DDoS) attacks, safeguarding the server's availability and performance. This proactive approach to security involves crafting rulesets that mitigate the impact of potential DDoS attempts.

As part of my ongoing homelab projects, I've made a old but good computer , transforming it into a dedicated firewall. Configuring OPNsense on this system allows me to explore the complexity of firewall management and network segmentation. Through this hands-on experience, I gain valuable insights into the functionality of firewalls and their role in network security.

In the homelab environment, I actively engage in network segregation exercises, experimenting with different configurations to deepen my understanding of network architecture. This approach enables me to get a grip of network segmentation and its impact on overall security.

Taking my commitment a step further, I've initiated penetration testing on the server. This involves simulating various attack scenarios to identify potential vulnerabilities. Subsequently, I dedicate time to harden the server based on the insights gained from these self-conducted penetration tests. This iterative process not only fortifies the server against potential threats but also enhances my knowledge of security best practices.

Overall, my exploration of Proxmox, home server setup, and involvement in homelab projects constitute a holistic journey toward mastering server management and cybersecurity. Through continuous learning and practical experimentation, I strive to create a robust and secure computing environment. I've been thinking of this like a cat and mouse game with myself where i try to find loop holes and also patch these sores while making the server working and still being able to host this site.
